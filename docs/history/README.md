## How to use these

Use history docs to understand **why** a decision was made, not **what** the current state is. Current state lives in the operations docs.

If you need to know what migrations were written in a sprint, these reports are the fastest path. If you need to know the current migration dependency order, use [operations/runbook.md](../operations/runbook.md).
